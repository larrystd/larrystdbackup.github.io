muduo网络库简介

概述

STL算法部分主要由头文件<algorithm>,<numeric>,<functional>组成。要使用 STL中的算法函数必须包含头文件 <algorithm>，对于数值算法须包含 <numeric>，<functional> 中则定义了一些模板类，用来声明函数对象。

算法

查找算法 #include<alogorithm>

  • ForwardIt adjacent_find( ForwardIt first, ForwardIt last ): 在iterator对标识元素范围内，查找一对相邻重复元素，找到则返回指向这对元素的第一个元素的ForwardIterator。否则返回last。重载版本使用输入的二元操作符代替相等的判断。

  • count( InputIt first, InputIt last, const T &value ): 利用等于操作符，把标志范围内的元素与输入值比较，返回相等元素个数。

  • count_if( InputIt first, InputIt last, UnaryPredicate p ): 利用输入的操作符，对标志范围内的元素进行操作，返回结果为true的个数。

  • bool binary_search( ForwardIt first, ForwardIt last, const T& value ): 在有序序列中查找value，找到返回true。重载的版本实用指定的比较函数对象或函数指针来判断相等。

  • std::pair<ForwardIt,ForwardIt> equal_range( ForwardIt first, ForwardIt last,const T& value );: 功能类似equal，返回一对iterator，第一个表示lower_bound，第二个表示upper_bound。
    find: 利用底层元素的等于操作符，对指定范围内的元素与输入值进行比较。当匹配时，结束搜索，返回指向该元素的Iterator。

  • InputIt find( InputIt first, InputIt last, const T& value ) 返回找到的第一个元素的迭代器。

  • InputIt find_if( InputIt first, InputIt last,UnaryPredicate p): 使用输入的函数代替等于操作符执行find。

  • ForwardIterator1 search (ForwardIterator1 first1, ForwardIterator1 last1,ForwardIterator2 first2, ForwardIterator2 last2,BinaryPredicate pred);: 给出两个范围，返回一个ForwardIterator，查找成功指向第一个范围内第一次出现子序列(第二个范围)的位置，查找失败指向last1。重载版本使用自定义的比较操作。

  • ForwardIterator search_n ( ForwardIterator first, ForwardIterator last,Size count, const T& val, BinaryPredicate pred );: 在指定范围内查找val出现n次的子序列。重载版本使用自定义的比较操作。

迭代器

链表迭代器示例

template <class Item>
struct ListIter {
    Item* ptr;  // ptr可以操作容器对象，即链表
    ListIter (Item* p = 0)
        : ptr(p) { }
    
    Item& operator*() const {
        return *ptr;
    }
    Item* operator->() const {
        return ptr;
    }

    ListIter& operator++() {
        ptr = ptr->next();
        return *this;
    }

    bool operator==(const ListIter& i) const {
        return ptr == i.ptr;
    }
}

Reactor模式

non-blocking IO+IO multiplexing这种模型，程序的基本结构是一个事件循环(event loop)，以事件驱动(event-driven)和事件回调的方式实现业务逻辑。

one loop per thread

此种模型下，程序里的每个IO线程有一个event loop，用于处理读写和定时事件。

Eventloop代表线程的主循环，需要哪个线程，就把timer或IO channel注册到哪个线程的loop里面(loop接受std::function作为执行的任务)。

推荐的C++多线程服务器编程模式是: one loop per thread + thread pool.

  • event loop(也叫IO loop)用做IO multiplexing，配合non-blocking IO和定时器
  • thread pool用来做计算，具体可以是任务队列或生产者消费者队列。

进程间通信只用TCP

Linux下进程间通信(IPC)，可以有匿名管道(pipe)，具名管道(FIFO)，POSIX消息队列，共享内存，信号(signals)等。同步原语(synchronization primitives)也很多，例如互斥锁(mutex)，条件变量(condition variable), 读写锁(reader-writer lock),文件锁(record locking),信号量(semaphore)等

进程间通信首选sockets, tcp socket可以read/write/fcntl/select/poll等，可以跨主机。

tcp port由一个进程独占，且操作系统会自动回收。listening port和已建立链接的TCP socket都是文件描述符，进程结束时操作系统会关闭所有文件描述符。因此即使意外退出，也不会留下垃圾。port独占可以防止程序重复启动。

可记录可重现，两个进程tcp通信一个崩溃，操作系统关闭连接，另一个进程几乎立刻感知。tcpdump和wireshark是解决状态问题的好帮手。tcp连接还可以跨语言。客户端和服务端不必一种语言。
