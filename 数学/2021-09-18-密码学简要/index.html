AES

AES(Advanced Encryption Standard), 又称Rijndael加密法
AES中s-box生成, 行移位, 列混淆, 轮密钥加等都是可逆的, 仿射变换(乘矩阵)是可逆的, 因此解密就是逆运算。

流程

  • AES一共10轮，每一轮的基本流程是字节代换, 行移位, 列混淆, 轮密钥加、

  • 字节代换, 将上一轮输出结果作为位置, 对s-box值进行代换

  • 行移位, 第1,2,3,4行分别不移动, 左移1, 左移2, 左移3。目的是扩散

  • 列混淆, 遵循伽罗瓦域乘法, 在GF(2^8)中

列混淆的乘积多项式

乘法逻辑遵循有限域乘法逻辑。

  • 轮密钥加(密钥只在这里起作用), 按位置与密钥值进行异或操作。

s-box

字节代换是将明文用s-box的对应元素代换, s-box不是随机生成的, 而是特定的。换句话说, 所有AES加密用的s-box都是一致的。

s-box的生成大致分为两步

  1. 找关于GF(2^8)的乘法逆元, GF(2^8)的素多项式可以表示为x^8+x^4+x^3+x+1, 也就是1 0001 1011。换句话说, 找明文模1 0001 1011的乘法逆元。这步可用扩展欧几里得算法完成。
  2. 将乘法逆元进行一个仿射变换

    注意最后异或的多项式, 0110 0011也就是63, 正好是00对应的s-box值。

  1. 注意以上是在GF(2^8)伽罗瓦域(有限域)中进行的, 乘除操作遵循有限域多项式, 加减操作变成异或。包括扩展欧几里得求解也是伽罗瓦域的求解。
  • 多项式除法

如上, 1000 1000 0100在GF(2^8) 除1 0001 1011的结果为1 0001, 余数为0010 1111。

乘法逆元

  • 欧几里得算法, 欧几里得算法的目的是找出两个整数的最大公因数, 即gcd(a,b)=gcd(b, a mod b)。
int gcd(int r0, int r1)
{
    int r=0;
    while(r1 != 0)
    {
        r = r0 % r1;
        r0 = r1;
        r1 = r;
    }
    return r0;
}
  • 扩展欧几里得算法, 扩展欧几里得算法的作用是给定整数r0,r1, 它计算如下公式, 其中s,t均为整数。

当gcd(r0,r1)=1时, 这个公式等价于t*r1 = 1 mod r0, 因此求解得到的t是r1 mod r0的乘法逆元。

在s-box生成时, 第一步就是计算乘法逆元。r0是GF(2^8)的素多项式1 0001 1011， 当输入值r1=1001时, 也就是求1001 mod 1 0001 1011的乘法逆元, 这可以用扩展欧几里得算法求解。

所谓素多项式, 也就是和域内其他多项式表征的数互素。一般都是指定的

一般的扩展欧几里得算法求逆元

int EEA(int r0, int r1)
{
    int mod = r0;
    int r = 0;
    int t0 = 0;
    int t1 = 1;
    int t = t1;
    int q = 0;
 
    //0不存在乘法逆元
    if (r1 == 0)
    {
        return 0;
    }
 
    while (r1 != 1)
    {
        q = r0 / r1;
 
        r = r0 - q * r1;
 
        t = t0 - q * t1;
 
        r0 = r1;
        r1 = r;
        t0 = t1;
        t1 = t;
    }
 
    //结果为负数
    if (t < 0)
    {
        t = t + mod;
    }
 
    return t;
}

GF(2^8)上的扩展欧几里得算法求逆元

//获取最高位
int GetHighestPosition(unsigned short Number)
{
    int i = 0;
    while (Number)
    {
        i++;
        Number = Number >> 1;
    }
    return i;
}
 
 
//GF(2^8)的多项式除法
unsigned char Division(unsigned short Num_L, unsigned short Num_R, unsigned short *Remainder)
{
    unsigned short r0 = 0;
    unsigned char  q = 0;
    int bitCount = 0;
 
    r0 = Num_L;
 
    bitCount = GetHighestPosition(r0) - GetHighestPosition(Num_R);
    while (bitCount >= 0)
    {
        q = q | (1 << bitCount);
        r0 = r0 ^ (Num_R << bitCount);
        bitCount = GetHighestPosition(r0) - GetHighestPosition(Num_R);
    }
    *Remainder = r0;
    return q;
}
 
 
 
//GF(2^8)多项式乘法
short Multiplication(unsigned char Num_L, unsigned char Num_R)
{
    //定义变量
    unsigned short Result = 0;      //伽罗瓦域内乘法计算的结果
 
    for (int i = 0; i < 8; i++)
    {
        Result ^= ((Num_L >> i) & 0x01) * (Num_R << i);
    }
 
    return Result;
}
 
/// 伽罗瓦域的扩展欧几里得
int EEA_V2(int r0, int r1)
{
    int mod = r0;
    unsigned short r = 0;
    int t0 = 0;
    int t1 = 1;
    int t = t1;
    int q = 0;
 
    if (r1 == 0)
    {
        return 0;
    }
 
    while (r1 != 1)
    {
        //q = r0 / r1;
        /// 伽罗瓦域除法
        q = Division(r0, r1, &r);
        /// 加减法变成异或
        r = r0 ^ Multiplication(q, r1);
        /// 
        t = t0 ^ Multiplication(q, t1);
 
        r0 = r1;
        r1 = r;
        t0 = t1;
        t1 = t;
    }
 
    if (t < 0)
    {   
        /// 加减法变异或
        t = t ^ mod;
    }
 
    return t;
}

求完逆元之后再执行依次仿射变换就可以得到对应s-box的值, 不再赘述。注意有限域元素乘法为&, 加为^,其他和矩阵乘法逻辑一致。

必要数论

欧拉函数

欧拉函数指任意给定正整数n，在小于等于n的正整数之中与n互质的个数

常用

  • n是质数，则 $φ(n)=n-1$
  • 如果n可以分解成两个互质的整数之积, $φ(n) = φ(p1p2) = φ(p1)φ(p2)$
  • a的 $φ(n)-1$ 次方，就是a的模n的逆。

素根

素根也叫原根, 若一个数g是P的原根，那么$g^i mod p$的结果两两不同，且有 1<g<p，0<i<p，归根到底就是$g^(P-1) = 1 (mod P)$当且仅当指数为P-1的时候成立.(这里P是素数)。

非对称加密

RSA

  • 密钥生成
  1. 随机选择两个不相等的质数p和q
  2. 计算p和q的乘积n。
  3. 计算n的欧拉函数φ(n), 其中　$φ(n) = (p-1)(q-1)$
  4. 随机选择一个整数e，条件是1< e < φ(n)，且e与φ(n) 互质
  5. 计算e对于φ(n)的模反元素d, 即

$e^d ≡ 1 (mod φ(n))$, 可以使用扩展欧几里得算法求解

  1. 将n和e封装成公钥，n和d封装成私钥

思路, 很难在已知n和e的情况下，推导出d。也就是说如果n可以被因数分解(n = p*q, φ(n) = (p-1)(q-1), 已知φ(n)就可以通过扩展欧几里得算法求d, 问题是求不出φ(n), 或者说p,q)，因为大整数的因数分解，是一件非常困难的事情。

  • 加密用公钥 (n,e)

$m^e ≡ c (mod n)$, 把c发出

  • 解密用私钥(n,d)

$c^d ≡ m (mod n)$, 得到信息m

ElGamal 算法

如同RSA算法基于大整数很难求解它的因式分解, ElGamal 算法基于离散对数问题很难被求解

  • 密钥生成

  1. 取一个随机的素数 p以及它的一个素根g

  2. 随机取一个小于p的数x，Private Key 为 <p,x>，

  3. 根据x算出 $y=g^x(mod p)$，Public Key 为 <p,g,y>

  • 加密, 随机选取整数r, 计算
    $C_1=g^r(mod p)$

$C2=m∗y^r(mod p)$, m为明文

发送(C1,C2)

  • 解密, 接收到了(C1,C2)

利用公式$C_2 / (C_1^x) (mod p) $得到明文, 可以令$K=(C_1^x) (mod p)$, $m = C2 * K^{-1} (mod p)$, $K^{-1}$为$K mod p$的逆。

Diffie-Hellman密钥交换算法

  • 前提是Alice和Bob已经达成$p$, $g$的一致, $P$为大素数, $g$为其的素根。

  • Alice 生成一个随机数 a, Alice 把 $y_A=g^a(mod p)$ 发送给Bob;

  • Bob 生成一个随机数 b, Bob 把 $y_B=g^b(mod p)$ 发送给Alice;

  • 双方达成共同的密钥
    $K_{AB}=y_B^a(mod p)=(g^b)^a(mod p) = (g^a)^b(mod p)$

  • 已知$y_A$, $y_B$, 很难推导得到$y_{AB}$, 也就是$g^{ab} (mod p)$
