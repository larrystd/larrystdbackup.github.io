可调用对象

七种可调用对象

  • 用户定义的函数，使用def或lambda表达式创建
  • 内置函数，使用C语言实现的函数，如len
  • 内置方法，用C语言实现的方法
  • 方法，类中的方法
  • 类，调用类时会运行__new__产生一个实例，然后运行__init__方法初始化，最后把实例返回给调用方
  • 类的实例，如果类定义__call__方法，实例可以作为函数调用
  • 生成器函数，使用yield关键字的函数或者方法，生成器函数返回的是生成器对象

可以调用内置的callable()函数判断对象能否被调用

任何python对象都可以表现的像函数，只需要实现实例方法__call__, 作用相当于cpp的operator()

import random

class BingoCage:
    def __init__(self, items):
        self._items = list(items)
        random.shuffle(self._items)
    def pick(self):
        try:
            return self._items.pop()
        except IndexError:
            raise LookupError('pick from empty BingoCage')
    def __call__(self):
        return self.pick()

bingo = BingoCage(range(3))
# 作为函数使用
bingo()

class Average():
    def __init__(self):
        self.series = []

Cpython垃圾回收主要算法是引用计数，每个对象都会统计多少引用指向自己，当引用计数归零对象会被立即销毁。Cpython调用对象的__del__方法，释放分配给对象的内存。Cpython2增加了分代垃圾回收算法。

有时候需要引用对象而不让对象存在时间超过所需要时间，这时候用弱引用，类似于cpp的weak_ptr。可以解决shared_ptr循环引用的问题，如果需要调用将weak_ptr提升为shared_ptr，不用了就reset shared_ptr，也可以避免对象一只被shared_ptr而无法销毁的问题。

一个简单的装饰器，输出函数运行时间

装饰器，用看作函数的函数，一旦调用某个函数，自动调用装饰器模块执行一些计算时间之类的工作

# clockdeco.py
import time

def clock(func):
    def clocked(*args):
        t0 = time.perf_counter()
        result = func(*args)
        elapsed = time.perf_counter() - t0
        name = func.__name__
        arg_str = '. '.join(repr(arg) for arg in args)
        print('[%0.8fs] %s(%s) -> %r' % (elapsed, name, arg_str, result))
        return result
    return clocked

# demo.py
import time
from clockdeco import clock

@clock
def snooze(seconds):
    time.sleep(seconds)

@clock
def factorial(n):
    return 1 if n < 2 else n*factorial(n-1)

if __name__ == '__main__':
    print('*'*40, 'Calling snooze(.123)')
    snooze(.123)
    print ('*'*40, 'Calling factorial(6)')
    print('6!= ', factorial(6))

进一步装饰器

python的装饰器十分强大，@classmethod, @staticmethod, @property等内部实现均是装饰器。

  • @classmethod定义类方法,@staticmethod是静态方法。二者的使用基本一致，只不过@classmethod逻辑上属于该类，@staticmethod与类无依赖关系，只是通过类调用而已, 与普通方法无异。
class A(object):
    def m1(self, n):
        print("self:", self)

    @classmethod
    def m2(cls, n):
        print('classmethod')
        print("cls:", cls)

    @staticmethod
    def m3(n):
        print('staticmethod')

a = A()
a.m1(1) # self: <__main__.A object at 0x000001E596E41A90>
A.m2(1) # cls: <class '__main__.A'>
A.m3(1)
  • @property修饰方法，将方法用属性形式调用，但不能修改只能获取，起到对私有变量的getter的作用。
class Vector2d:
    typecode = 'd'

    def __init__(self, x, y):
        self._x = float(x)
        self._y = float(y)
    @property
    def x(self):
        return self._x
    def __iter__(self):
        return (i for i in (self.x, self.y))

Python中不存在只能从对象内部访问的“私有(private)”变量，但Python制定了以下公约：

  • 以单下划线开头的成员变量或方法，是设计者不想暴露给外部的API，使用者不应该进行访问。但类实例和子类实例能访问到这些变量
  • 以双下划线开头且以至多一个下划线结尾的成员变量或方法，会进行名称转写。可以通过名称改写访问到

python中的私有变量和私有方法仍然是可以访问的；访问方法（名称改写）如下：

  • 私有变量:实例._类名__变量名
  • 私有方法:实例._类名__方法名()

可迭代 和生成器

所有生成器都是迭代器，因为生成器完全实现了迭代器的接口。迭代器用来从集合中取元素，生成器则用于生成元素。

import re
import reprlib

RE_WORD = re.compile('\w+')

class Sentence:
    def __init__(self, text):
        self.text = text
        self.words = RE_WORD.findall(text)
    def __getitem__(self, index)
        return self.words[index]
    def __len__(self):
        return len(self.words)
    def __repr__(self):
        return 'Sentence(%s)' % reprlib.repr(self.text)

s = Sentence('The time has come')
for word in s:
    print(word)

list(s)

re.finall函数返回正则表达式全部非重叠匹配后的字符串列表。self.words为字符串列表结果。

解释器需要迭代对象x时，会自动调用iter(x)。如果对象实现了__iter__就调用之，若没有实现但实现了__getitem__方法，python会创建一个迭代器按顺序获取元素。

标准的迭代器接口有两个办法

  • __next__ 返回下一个可用的元素，如果没有元素，抛出StopIteration异常
  • __iter__ 返回self，以便在该应用迭代器地方使用迭代器。

class SentenceIterator:
    def __init__(self, words):
        self.words = words
        self.index = 0
    def __next__(self):
        try:
            word = self.words[self.index]
        except IndexError:
            raise StopIteration()
        self.index += 1
        return word
    def __iter__(self):
        return self

生成器到协程

Cpython解释器本身不是线程安全的，因此有GIL: 全局解释器锁（英语：Global Interpreter Lock，缩写GIL），它使得任何时刻仅有一个线程在执行。[1]即便在多核心处理器上，使用 GIL 的解释器也只允许同一时间执行一个线程。由于GIL的存在，导致Python多线程性能甚至比单线程更糟。
但python线程等待操作系统返回结果时会释放GIL，例如等待网络响应，这时其他线程可以执行，意味着python多线程能获得些许提升。

于是出现了协程（Coroutine）。协程的作用，是在执行函数A时，可以随时中断，去执行函数B，然后中断继续执行函数A（可以自由切换）。但这一过程并不是函数调用（没有调用语句），这一整个过程看似像多线程，然而协程只有一个线程执行.

如上，方法中包含yield表达式后，Python会将其视作generator对象，不再是普通的方法。生成器这种执行一次就阻塞等待__next__的特性可用作协程。

def test():
    print("generator start")
    n = 1
    while True:
        yield_expression_value = yield n
        print("yield_expression_value = %d" % yield_expression_value)
        n += 1


# ①创建generator对象
generator = test()
print(type(generator))

print("\n---------------\n")

# ②启动generator
next_result = generator.__next__()
print("next_result = %d" % next_result)

print("\n---------------\n")

# ③发送值给yield表达式
send_result = generator.send(666)
print("send_result = %d" % send_result)

<class 'generator'>
---------------
generator start

---------------

启动generator后，代码将执行到yield出现的位置，也就是执行到yield n，然后将n传递到generator.next()这行的返回值。生成器执行到yield n后将暂停在这里，直到下一次生成器被启动。

调用send方法可以发送值给yield表达式，同时恢复生成器的执行。生成器从上次中断的位置继续向下执行，然后遇到下一个yield，生成器再次暂停，切换到主函数打印出send_result。

生产者，消费者模型。模式是生产一个，消费一个，这不会阻塞，而是线程自动执行一次producer，再执行一次comsumer,循环往复。

def consumer():
    print("[CONSUMER] start")
    r = 'start'
    while True:
        n = yield r
        if not n:
            print("n is empty")
            continue
        print("[CONSUMER] Consumer is consuming %s" % n)
        r = "200 ok"


def producer(c):
    # 启动generator
    start_value = c.send(None)
    print(start_value)
    n = 0
    while n < 3:
        n += 1
        print("[PRODUCER] Producer is producing %d" % n)
        # 唤醒yield
        r = c.send(n)
        print('[PRODUCER] Consumer return: %s' % r)
    # 关闭generator
    c.close()


# 创建生成器
c = consumer()
# 传入generator
producer(c)

下载

import os
import time
import sys

import requests

POP20_CC = ('CN IN US ID BR PK NG BD RU JP').split()
BASE_URL = 'http://flupy.org/data/flags'
DEST_DIR = 'downloads/'

def save_flag(img, filename):
    path = os.path.join(DEST_DIR, filename)
    with open(path, 'wb') as fp:
        fp.write(img)

def get_flag(cc):
    url = '{}/{cc}/{cc}.gif'.format(BASE_URL, cc = cc.lower())
    resp = requests.get(url)
    return resp.content

def show(text):
    print(text, end=' ')
    sys.stdout.flush()

def download_many(cc_list):
    for cc in sorted(cc_list):
        image = get_flag(cc)
        show(cc)
        save_flag(image, cc.lower()+'.gif')
    return len(cc_list)

def main(download_many):
    t0 = time.time()
    count = download_many(POP20_CC)
    elapsed = time.time() - t0
    msg = '\n{} flags downloaded in {:.2f}s'
    print (msg.format(count, elapsed))

if __name__ == "__main__":
    main(download_many)
