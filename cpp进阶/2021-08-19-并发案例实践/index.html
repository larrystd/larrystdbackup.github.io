所有的并发都是被管理，有消息传递的局部并行。异步执行不意味着进程放任不管了, 至少需要进行在执行到某个地方返回有效的信息。

并发编程需要思考的是进程的协调，互相调用

生产者消费者模型

这里使用mutex和condition_variable来协调线程。

主要记住,

  • wait条件不满足之后会释放锁等待, 以后必须被其他线程唤醒才会向下执行, 而不是等待条件满足了。
  • 使用条件变量, wait和notify_all要同时用, 且notify_all之前需要释放锁。这时候我们可以选择手动释放锁(lock_guard不能手动释放), 或用{}设置块作用域
  • 为了能够缓慢复现, 一定给予sleep时间, 也就是std::this_thread::sleep_for(dura)
#include <iostream>
#include <mutex>
#include <thread>
#include <condition_variable>
#include <queue>

using namespace std;

class ComsumerDemo {
public:
    void comsumer() {
        while (true) {
            //std::lock_guard<std::mutex> lcg(mtx);
            {
                std::unique_lock<std::mutex> lck(mtx);
                ctx.wait(lck, [this]() {
                    return !q.empty();
                });
                //while (q.empty()) {
                //	ctx.wait(lck);	// wait unique_lock 而不是mutex
                //}
                int data = q.front();
                q.pop();
                printf("consume %d\n", data);
                cout << "consume q.size="<< q.size() <<endl;
            // 在这里unique_lock锁会自动释放
            }
            
            ctx.notify_all();
            /* 等待2S */
            // 一定留足时间
			std::chrono::milliseconds dura(2000);
			std::this_thread::sleep_for(dura);
        }      
    }
    void productor() {
        while (true) {

            std::unique_lock<std::mutex> lck(mtx);
            num = (num+1)%10;
            int n = q.size();
            ctx.wait(lck, [this, n]() {
                cout << "q.size="<< n <<endl;
                return n < 10;
            });

            //while (q.size() >= 10) {
            //    ctx.wait(lck);
            //}
            q.push(num);
            printf("produce %d\n", num);
            lck.unlock();   // 手动释放锁
            ctx.notify_all();

            /* 等待1S */
			std::chrono::milliseconds dura(1000);
			std::this_thread::sleep_for(dura);
        }
    }

    void start() {
        // 创建后自动执行函数
        std::thread t1(&ComsumerDemo::comsumer, this);
        std::thread t2(&ComsumerDemo::productor, this);
        //std::thread t3(&ComsumerDemo::comsumer, this);

        t1.join();
        t2.join();
        //t3.join();
    }
private:
    std::mutex mtx;
    std::condition_variable ctx;
    queue<int> q;
    int num;
};

异步等待模型

使用future可以轻易实现异步等待。即线程2等待线程1执行后立即执行，依次类推。

原因在于std::future<T>::wait 和 std::future<T>::get 都会阻塞地等待拥有它的 promise 对象返回其所存储的值，后者还会获取 T 类型的对象；

如leetcode 1114

class Foo {
    promise<void> pro1, pro2;

public:
    void first(function<void()> printFirst) {
        printFirst();
        pro1.set_value();
    }

    void second(function<void()> printSecond) {
        /// 下面这句话相当于等待pro1执行完set_value()再运行
        pro1.get_future().wait();
        printSecond();
        pro2.set_value();
    }

    void third(function<void()> printThird) {
        pro2.get_future().wait();
        printThird();
    }

    void start() {
        std::thread t1 (&Foo2::first, this,[]() {
            cout << "first" << endl;
        });

        std::thread t2 (&Foo2::second, this,[]() {
            cout << "second" << endl;
        });

        std::thread t3 (&Foo2::third, this,[]() {
            cout << "third" << endl;
        });

        t1.join();
        t2.join();
        t3.join();
    }
};

原子模型

将共享标志变量设置为原子变量，从而在一个线程执行完修改标志位，使另一个线程执行。

class Foo {
    std::atomic<bool> a{ false };
    std::atomic<bool> b{ false };
public:
    void first(function<void()> printFirst) {
        printFirst();
        /// 修改标识变量
        a = true;
    }

    void second(function<void()> printSecond) {
        /// 可以直接循环等待标识变量的修改
        while (!a)
            this_thread::sleep_for(chrono::milliseconds(1));
        printSecond();
        b = true;
    }

    void third(function<void()> printThird) {
        while (!b)
            this_thread::sleep_for(chrono::milliseconds(1));
        printThird();
    }
};
