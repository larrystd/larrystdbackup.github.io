函数模板

以下是一个函数模板，返回两个数之间的最大值。T是类型参数。

template <typename T>
T max (T a, T b){
    return b < a ? a : b;
}

两阶段编译检查

模板分两步检查，

  1. 模板定义阶段，不包含诶行参数的检查。
  2. 模板实例化阶段，模板会再次被检查，尤其是依赖于类型参数的部分。
template<typename T>
void foo(T t){
    undeclare();    // 若函数未定义，第一阶段报错
    undeclare(t);   // 如果参数不对，第二阶段报错
}

以上可以总结为，和模板参数有关的，总是在第二阶段被检查。

类型推断

  1. 调用参数按引用传递，任何类型转换都不允许。通过模板类型T定义的两个参数，它们的实参必须完全一样。
  2. 调用参数按值传递，只有退化被运行，const和volatile限制符会被忽略。引用转化成被引用的类型，raw array和函数被转换成相应的指针类型。T定义的两个参数，实参退化后类型必须一样。
    例
template<typename T>
T max (T a, T b);

int& ir = i;
max(i, ir); // 被推导成int, 引用被decay
int arr[4];
foo(&i, arr);   // 被推导成int*
// 以下错误
max(4, 7.2);    // 错误，不确定被推导为int还是double

有两种办法解决以上错误

1. 对参数做类型转换
max(static_cast<double>(4), 7.2);   
2. 显示指出参数T的类型，编译器不再进行类型推导
max<double> (4,7.2);
3. 设置不同模板参数

默认调用参数

template<typename T>
void f(T = "");

f(1);   // ok,调用f<int> (1)
f();    // 错误，无法推断T的类型

应对这一情况，增加一个默认参数

template<typename T = std::string>
void f(T = "")
f();    // ok

返回类型推断

C++11 中，尾值返回类型（trailing reture type)允许使用函数的调用参数，声明返回类型。

template<typename T1, typename T2>
auto max (T1 a, T2 b) -> decltype(b < a ? a : b)
{
    return b < a ? a : b;
}

但需要注意，T1可能是引用因此会推导出返回值为引用，但我们想让它返回退化后的T

#include <type_traits>
template<typename T1, typename T2>
auto max (T1 a, T2 b) -> typename std::decay<decltype(true ? a : b)> :: type

非模板类型参数

c++17开始，可用auto来设置任意有效的非类型模板参数

#include <array>
#include <cassert>

template<typename T, auto Maxsize>
class Stack {
    public:
    using size_type = decltype<Maxsize>;
    private:
    std::array<T, Maxsize> elems;
    size_type numElems;
}

一些基础技巧

typename关键字用来声明类型

typename T::subType* ptr

typename T::const_iterator pos;

以上typename用来澄清subType是定义在class T的一个类型，而不是数据成员。如果没有typename，subType会被假设成一个非类型成员。

零初始化

对于基础类型，比如int,指针类型，由于它们没有默认构造函数，因此它们不会被初始化成一个有意义的值。

int x;  // x has undefined value
int *ptr;   // ptr point to anywhere

对于内置类型，最好显示调用其默认构造函数初始化，比如

template<typename T>
void foo()
{
    T x{};
}

template<typename T>
class MyClass {
private:
    T x;
public:
    MyClass() : x{}
    // ensures x is initizlized even for built-in tboidype
}

继承

对于类模板，如果它的基类也依赖于模板参数，那么this->x和x不一定等效。

template<typename T>
class Base {
public:
    void bar();
};
template<typename T>
class Derived : Base<T>{
public: 
    void foo() {
        bar();  // call external bar or error() 
    }
}

以上Derived中的bar永远不会被解析成Base中的bar,作为经验法则，建议用this->或base<T>::修饰之。

值传递和引用传递

按值传递

当按值传递参数时，原则上所有的参数都会被拷贝，但注意被传递的参数是左值lvalue，拷贝构造函数会被调用。但是传递参数是右值，编译器会优化参数传递，使拷贝构造函数不会被调用。

按值传递会导致类型退化decay，裸数组退化成指针，const和volatile等限制符被解除。

template<typename T>
void printV (T arg){
    ...
}

std::string const c = "hi";
printV(c);  // c decays so that atg type is std::string
printV("hi"); // char const[3] decays to pointer, char const*
int arr[4];
printV(arr); // decays to pointer arg type int*

按引用传递

引用传递类型不会退化，也就是裸数组不会转换成指针，也不会移除const和volatile等限制符。

template<typename T>
void printR(T const& arg){

}
std::string const c = "hi";
printR(c);  // 参数，std::string const&
int arr[4];
printR(arr);    // 参数，int const(&) [4]

按转发引用传递参数

template<typename T>
void passR (T && arg) {

}


std::string s = "hi";
passR(s);   // T std::string&, arg is also
passR(std::string("hi")); // T std::string, arg std::string&&

passR(std::move(s)); // T std::string arg is std::string&&
passR(arr); // T int(&)[4] also the type of arg

综上，左值与通用引用放在一推导出来的T仍为左值，而右值与通用引用放在一起推导出来的T仍然为右值

动态多态和静态多态

动态多态

多态设计的艺术性主要体现在从一些相关的对象类型中提炼出一组统一的功能，然后将它们声明成一个基类的虚函数接口。而特定的功能则从它做了继承。

#include "coord.hpp"

class GeoObj {
    public:
    virtual void draw() const = 0;
    virtual Coord center_of_gravity() const = 0;
    virtual ~GeoObj() = default;
};

class Circle : public GeoObj {
    public:
    virtual void draw() const override;
    virtual Coord center_of_gravity() const override;
};

class Line : public GeoObj {
    public:
    virtual void draw() const override;
    virtual Coord center_of_gravity() const override;
};

Coord distance (GeoObj const& x1, GeoObj const& x2)
// 传引用，可以保护x1,x2的动态类型
Line x1,x2;
distance(x1,x2); // distance(GeoObj&, GeoObj&)

静态多态

不再使用虚函数层级，而是独立的几何类，如下


class Circle{};
class Line{};

template <typename GeoObj>
void myDraw (GeoObj const& obj){
    obj.draw();
}

template<typename GeoObj1, typename GeoObj2>
Coord distance (GeoObj1 const& x1, GeoObj2 const& x2){}

Circle c1,c2;
distance(c1,c2); // distance<Circle, Circle>
Line l;
distance(l, c1); // distance<Line, Circle>(GeoObj1&, GeoObj2&)

通常静态多态比动态多态更加类型安全，因为其所有的绑定都在编译期进行了检查。
